MiniDapp Development

Build decentralized apps with standard web technologies

What is a MiniDapp?

A MiniDapp is a decentralized web application that runs on a Minima node. Built with HTML, CSS, and JavaScript — no special framework required. Every MiniDapp gets sandboxed access to the blockchain via the MDS API.

Structure

my-minidapp/
├── dapp.conf       ← metadata (name, version, icon)
├── favicon.ico     ← app icon
├── index.html      ← main page
├── style.css       ← styles
├── service.js      ← background service (optional)
└── mds.js          ← MDS library

dapp.conf

{
  "name": "MyApp",
  "version": "1.0.0",
  "description": "What this app does",
  "icon": "favicon.ico"
}

MDS API

Initialize MDS in every page:

MDS.init(function(msg) {
  if (msg.event == "inited") {
    // MDS is ready — run commands here
  }
});

Key Functions

FunctionPurpose
MDS.cmd(command, cb)Run blockchain commands (balance, send, status)
MDS.sql(query, cb)Private H2 SQL database per MiniDapp
MDS.file.save/loadFile storage (text or binary)
MDS.keypair.set/getKey-value settings storage
MDS.api.call(dapp, data, cb)Call another MiniDapp's API
MDS.comms.broadcastSend messages to all MiniDapps
MDS.net.GET/POSTHTTP requests

Events

  • inited — MDS ready
  • NEWBLOCK — new block added to chain
  • NEWBALANCE — user balance changed
  • MINING — mining event

Packaging & Installing

Zip the folder contents (not the folder itself):

cd my-minidapp
zip -r ../myapp.mds.zip .

Install on mobile: Send zip to device → open Minima app → tap + → select file.

Install on desktop: MiniDapp Hub → click + → select zip. Or via CLI:

mds action:install file:/path/to/myapp.mds.zip

MiniDapp Hub

Access all installed MiniDapps at https://127.0.0.1:9003. MDS must be enabled (start with -mdsenable flag). Uses self-signed SSL certificates.
