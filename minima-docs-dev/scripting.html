KISS VM Scripting

Minima's Turing Complete smart contract language

Overview

Minima has its own Turing Complete scripting language for creating Smart Contracts — KISS VM (Keep It Simple Scripting).

Minima, like Bitcoin, uses the UTxO model so writing smart contracts is quite different to writing them on an account-based model like Ethereum. Every UTxO (coin) is associated with a script which can either return TRUE or FALSE when used in a transaction. The default is return FALSE, so all scripts must explicitly return TRUE for the transaction to be valid.

A "contract" is the script that locks the funds in a coin — the two terms are interchangeable.

Default Script

RETURN SIGNEDBY(0xFFEE67F7C..)

Where 0xFFEE67F7C.. is the public key of the owner. This script only returns TRUE when the transaction is signed by the coin's owner.

KISS VM supports SHA2-256 to allow cross-chain hash lock contracts with legacy chains. Contracts are inherently compatible with Layer 2.

Scripts are case sensitive. Users track coins at their own addresses and any coins referencing their keys in STATE or PREVSTATE.

State Variables

The addition of state variables in the MMR Proof DB allows for complex scripts with knowledge of their past. Minima uses a simple state mechanic for transactional history rather than a global state for ALL transactions (like Ethereum).

Each coin carries its own state in ports 0-255:

LET owner = STATE(0)
RETURN SIGNEDBY(owner)

Contract Types

Basic Signed

RETURN SIGNEDBY(0x)

Only the key holder can spend the coin.

Time Lock

IF @BLOCK GT 1000 THEN
  RETURN SIGNEDBY(0x)
ENDIF

Coin is locked until block 1000, then only the key holder can spend.

Multi-Sig (2 of 2)

RETURN SIGNEDBY(0x) AND SIGNEDBY(0x)

M-of-N Multi-Sig (2 of 3)

LET sigs = SIGNEDBY(0x) + SIGNEDBY(0x) + SIGNEDBY(0x)
RETURN sigs GTE 2

Hashed Time Lock Contract (HTLC)

Used for cross-chain atomic swaps. The recipient can claim with a preimage, or the sender reclaims after a timeout:

IF SIGNEDBY(0x) AND SHA2(STATE(0)) EQ 0x THEN
  RETURN TRUE
ELSEIF @BLOCK GT 5000 AND SIGNEDBY(0x) THEN
  RETURN TRUE
ENDIF

Exchange Contract

Trustless token swap — both parties must be present in the same transaction.

FlashCash

Instant micro-payments using pre-signed transactions.

MAST (Merkelized Abstract Syntax Trees)

Complex conditional logic where only the executed branch is revealed on-chain, keeping unused branches private.

Token Scripts

Each token has a separate script that must also return TRUE when attempting to spend a UTxO. A token by default has RETURN TRUE as its script. This structure is added to any transaction using that token so every user can verify the token details.

Both the coin address script AND the token script must return TRUE for a valid spend.

tokencreate name:MYTOKEN amount:1000 decimals:4 script:"RETURN SIGNEDBY(0x)"
