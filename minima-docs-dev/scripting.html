KISS VM Scripting

Minima's Turing Complete smart contract language

Overview

KISS VM (Keep It Simple Scripting) is Minima's scripting language. Unlike Ethereum's account model, Minima is UTxO-based — every coin carries a script that must return TRUE to be spent. Default is FALSE.

Limits

ParameterLimit
Max instructions1,024
Max stack depth64
Max function params32

Basic Scripts

Anyone Can Spend

RETURN TRUE

Only Owner Can Spend

RETURN SIGNEDBY(0x)

Time Lock

IF @BLOCK GT 1000 THEN
  RETURN SIGNEDBY(0x)
ENDIF

Multi-Sig (2 of 2)

RETURN SIGNEDBY(0x) AND SIGNEDBY(0x)

Contract Types

  • Basic Signed: Only the key holder can spend
  • Time Lock: Coins locked until a certain block height
  • Multi-Sig: Multiple signatures required (M-of-N)
  • Hashed Time Lock (HTLC): Cross-chain atomic swaps
  • Exchange: Trustless token swaps
  • MAST: Merkelized scripts for complex conditional logic
  • FlashCash: Instant micro-payments

State Variables

Scripts can read state variables stored in the MMR Proof DB. These are transactional (not global like Ethereum) — each coin carries its own state in ports 0-255.

LET owner = STATE(0)
RETURN SIGNEDBY(owner)

Token Scripts

Every token has its own script (separate from the coin script). Both must return TRUE for a valid spend. Default token script is RETURN TRUE.

tokencreate name:MYTOKEN amount:1000 decimals:4 script:"RETURN SIGNEDBY(0x)"
Scripts are case-sensitive. SHA2-256 is available for cross-chain hash locks. Contracts are inherently Layer 2 compatible.
