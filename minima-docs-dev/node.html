Getting Started

Run a node, secure it, and start using MiniDapps

Android

  1. Install from Google Play Store (or download APK from GitHub)
  2. Allow battery optimisation exemption for background operation
  3. Enable auto-update in Play Store settings
  4. Write down your seed phrase immediately
Huawei/Redmi/ZTE: Disable "Manage Automatically" in battery settings. OPPO: Keep background apps under 5.

Desktop Connect: Access your phone node from a computer on the same WiFi via Settings → Desktop Connect.

Desktop Command Line

Closing the terminal window or turning off your computer will stop Minima running. It is recommended to create a batch file (Windows) or bash script (Mac/Linux) to start Minima at startup.

Pre-requisites

1. Check you have Java installed

To run Minima, you need Java installed. Check by running:

java -version

If you don't have Java, download it from java.com.

2. Download the latest minima.jar

Download the latest version from the Minima GitHub releases page.

Start a new Minima node

  1. Create a new folder and copy the minima.jar into that folder. We will run everything from there.
  2. Open your command line interface / Terminal
  3. Copy the path to the folder and cd into it:

Windows:

cd C:\Users\Bob\MyMinimaFolder\

Mac/Linux:

cd ~/MyMinimaFolder/
Tip: To avoid typing the folder path, drag and drop the folder into the command line window.

Enter the following to start Minima with the MiniDapp System enabled. Replace INSERTPASSWORD with your own password:

java -jar minima.jar -data .minima -p2pnodes megammr.minima.global:9001 -mdsenable -mdspassword INSERTPASSWORD

Minima will start and automatically join the network. You should see the Minima ASCII logo and logs running in your terminal.

CLI only (no MiniDapps):

Mac:

sudo java -jar minima.jar -data .minima -p2pnodes megammr.minima.global:9001

Windows:

java -jar minima.jar -data .minima -p2pnodes megammr.minima.global:9001

Login to your node

Go to https://127.0.0.1:9003/ in your web browser. You may have to proceed past the security warning, since Minima uses self-signed certificates.

Enter the password you set as your -mdspassword when starting the node.

Custom Ports: 9003 is the default port for accessing the MiniDapp System. If running on custom ports, use your custom base port + 2. E.g. if you started on port 8001, use https://127.0.0.1:8003/
Forgot your password? Return to the terminal where the Minima logs are running and type mds to check your password.

Secure Your Node

Keeping your wallet secure involves:

  • Keeping the device your node is running on secure
  • Writing down your seed phrase immediately after starting your node
  • Locking your node with a password
  • Backing up your node

Your seed phrase is required to restore your wallet if you lose access to your node and do not have a backup.

How to find your seed phrase

Security MiniDapp:

  1. Open the Security MiniDapp and ensure your node is not locked
  2. Select Manage Seed Phrase
  3. Select Show Seed Phrase
  4. Grab a pen and paper and write down your Seed Phrase in the order specified

Command Line:

You can also view your seed phrase by running the vault command from the Minima Terminal, RPC client or Docker CLI.

Seed Phrase Best Practices

  • DON'T take a screenshot of your seed phrase
  • DON'T copy and paste your seed phrase to/from a device connected to the internet
  • DO write it on paper or use a metal plate to record it
  • DO make 2 copies and store them in separate locations
  • DO give one copy to a trusted relative
  • For large amounts, consider cold storage — signing transactions offline using one offline node and one online node

Using the MiniDapp Hub

By default, all MiniDapps are given READ permission. Any transactions or sensitive commands triggered from READ apps must be approved from the Pending MiniDapp.

MiniDapps with WRITE permission can transact or run any command without asking for your approval.

Warning: Before giving a MiniDapp WRITE permissions, ensure your wallet is locked with a password. Only give WRITE permission to MiniDapps that you trust.

Changing MiniDapp permissions

  1. Right click on a MiniDapp (or long press on mobile)
  2. Select Write mode (or Read mode)
  3. Click Confirm

How to Update Your Node

  1. Type quit into the terminal where the Minima logs are running
  2. Download the latest version of Minima from the GitHub releases page and replace the existing minima.jar with the new version
  3. Restart Minima using the same command as above
Warning: DO NOT use -clean when simply restarting your node — it will wipe your coins. Only use -clean if you want a completely fresh install.

How to Remove Your Node

Warning: Removing a node without taking a backup will result in losing access to your coins. Only remove a node if you have taken a backup or written down your 24-word seed phrase.
  1. Type quit into the terminal where the Minima logs are running
  2. Delete the minima.jar file you downloaded
  3. Remove the Minima database files located in your home directory (the .minima folder) or the directory you specified with -data. You may need to show hidden files.

Docker

docker run -d --name minima \
  -e minima_mdspassword=YOUR_PASSWORD \
  -e minima_desktop=true \
  -v minima-data:/home/minima/data \
  -p 9001-9003:9001-9003 \
  --restart unless-stopped \
  minimaglobal/minima:latest

Auto-update with Watchtower:

docker run -d --name watchtower \
  -v /var/run/docker.sock:/var/run/docker.sock \
  containrrr/watchtower \
  --interval 28800 minima

Login at https://localhost:9003 with your password.
