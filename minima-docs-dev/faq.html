Frequently Asked Questions

Common questions about developing on Minima

Standard web technologies: HTML, CSS, and JavaScript. React is also supported. No special framework required — if you can build a website, you can build a MiniDapp.
No. MiniDapps run directly on the user's Minima node. There is no central server, no database to manage, no hosting to pay for. Each user runs their own instance.
Start a local Minima node (CLI or Docker), install your MiniDapp via the Hub, and test at https://127.0.0.1:9003. For scripting, use the terminal to run commands directly. You can start a private testnet with 1 billion coins from genesis for safe testing.
KISS VM is UTxO-based (like Bitcoin Script but Turing Complete), while Solidity is account-based. In KISS VM, every coin carries its own script — the script answers one question: can this coin be spent in this transaction? State is per-coin, not global.
Yes. Use MDS.api.call("dappname", data, callback) to call another MiniDapp's API directly. Use MDS.comms.broadcast() to send messages to all installed MiniDapps. PocketWeb and PocketFS use this exact mechanism.
Run 'tokencreate name:MYTOKEN amount:1000 decimals:4' in the terminal. Tokens are native to the protocol — no smart contract deployment needed. NFTs are tokens with 0 decimals (indivisible). Each token gets a unique TokenID.
Minima uses the Winternitz One-Time Signature Scheme with SHA3-256 hashing — both considered quantum-resistant. Unlike ECDSA (Bitcoin/Ethereum), these cannot be broken by Shor's quantum algorithm. Each key is used only once for maximum security.
Four adaptive network parameters (max TxPoW size, max script operations, max transactions per block, min PoW work) that adjust over time via weighted consensus. This future-proofs the network as hardware improves — no hard forks needed.
