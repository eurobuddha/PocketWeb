Minidapp CLI

You can use the CLI to create a new Minima project with a template. The current templates that are available are:

  • react-ts
  • vanilla-js

init

Use the init command to initialize configuration and dependencies for a new project.

npm

```bash

npx @minima-global/create-minima-app init

<div class="tab-section"><h4>pnpm</h4>```bash
    pnpm dlx @minima-global/create-minima-app init

bun

```bash

bunx @minima-global/create-minima-app init

<div class="tab-section"><h4>yarn</h4>```bash
    yarn create @minima-global/create-minima-app init

You will be asked a few questions to configure your project to setup rpc and install your MiniDapp.

1. Whats the name of your project?
   2. Are you running Minima RPC?
   3. What port is your Minima node running on?
   4. Which template would you like to use?

options

You can pass the following flags to the init command to configure your project.

Initialize a new Minima App or configure an existing one

   Options:
   -n, --name <name>  the name of the app. (default: minima-app)
   -p, --port <port>  the port of the Minima node. (default: 9001)
   -r, --rpc <rpc>    the RPC URL of the Minima node. (default: true)
   -s, --service      create a service.js file. (default: false)
   -t, --template <template>  the template to use.
   -h, --help       display help for command

Minima CLI

The Minima CLI provides a set of scripts that can be used when developing a minidapp and include the following scripts:

  • zip
  • install
  • update
  • uninstall

Creating a project using create-minima-app includes the Minima CLI package automatically and provides a set of scripts located in your package.json file

{
 "minima:zip": "minima zip",
 "minima:install": "minima install",
 "minima:update": "minima update",
 "minima:uninstall": "minima uninstall",
 "minima:refresh": "minima zip && minima uninstall && minima install"
}
