The Blockchain

How Minima structures its chain for mobile-scale pruning

Overview

The Minima blockchain is structured to be compact and "lightweight" — it is heavily pruned to meet the requirement of running on mobile devices. The chain itself is a TxPoW Tree containing TxBlocks.

The blockchain has two parts:

  • TxPoW Tree — the main blockchain consisting of the most recent TxBlocks (approximately 1024 blocks kept in full)
  • Cascading Chain — an unbroken chain of the headers of "Super Blocks", designed to record and prove, in an immutable way, the total cumulative Proof-of-Work input into the network without having to keep a record of all individual blocks

The root of the TxPoW Tree always remains connected to the tip of the Cascading Chain. Blocks are added to the Cascading Chain on a periodic basis.

TxPoW Tree

The TxPoW tree is the main blockchain with a root (start), tip (end), blocks (tree nodes) and length (number of blocks).

Branching can occur if multiple blocks with the same height are found simultaneously. Branching is a natural occurrence due to network latency and will resolve over time as a heavier branch continues to be built on.

Each TxPoW Tree Node contains: the TxBlock itself, an MMR Set (constructed from the TxBlock), an array of all coins (spent and unspent), and an array of MMR entry numbers for coins relevant to the local user.

The Cascading Chain

The Cascading Chain records and proves the total cumulative Proof-of-Work input into the network without keeping all individual blocks. By identifying multiple levels of difficulty above the required block difficulty, "Super Blocks" emerge which provide orders of magnitude more PoW than typical blocks.

Key Properties

  • 32 levels (0-31), with a maximum of 128 blocks at each level
  • Grows logarithmically — each level is twice as difficult as the previous
  • Over time, the cumulative weight recorded tends towards what would have existed had no blocks been pruned
  • The chain is unbroken — each block references its previous super parent block
  • Updated every 100 block intervals

Block weight at level L = base weight × 2L. One Level 4 Super Block provides the PoW equivalent of sixteen Level 0 blocks.

Super Parents

Every block header contains references to a parent at every existing Super level in the cascade. When there are multiple blocks at a Super level, a new block only references the most recent one. This creates an immutable link structure ensuring the cascade is unbroken and independently verifiable.

Cascade Update Process

Once the main chain reaches 1124 blocks (1024 + 100), the cascading process begins:

  1. Level 0: The first 100 blocks from the main chain are added to Level 0 in the new cascade. 28 Level 0 blocks from the previous cascade fill the remaining 128 spaces.
  2. Level 1: After 128 blocks fill Level 0, remaining blocks are only kept if they meet Level 1 difficulty or above.
  3. Level 2+: The process continues — each level filters for progressively harder Super Blocks.
  4. The 101st block becomes the new root of the TxPoW Tree. Before pruning, its MMR Set is updated with CoinProofs for all unspent coins from the pruned blocks.

The result: a new Cascading Chain plus the most recent 1024 blocks kept in full on the main chain. CoinProofs are never lost.
