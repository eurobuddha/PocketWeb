Core Concepts

The fundamental building blocks of the Minima protocol

UTxO Model

Minima uses the Unspent Transaction Output model, like Bitcoin. Each Transaction Output is a specific amount of Minima with a unique Coin ID. Coins must be spent in whole — change is returned as a new UTxO.

The blockchain tracks all UTxOs in the network. The sum of all UTxO values equals at most 1 billion Minima. Transactions consume one or more UTxOs as inputs and create new UTxOs as outputs.

Example: Bob sends 30 Minima to Alice using a 50 Minima UTxO. Two new outputs are created: 30 to Alice and 20 returned to Bob as change.

Transaction Proof-of-Work (TxPoW)

Unlike Bitcoin where dedicated miners do all the work, Minima requires every user to mine their own transactions. This involves ~1 second of hashing before a transaction can be broadcast.

  • There are no block rewards and no fee collection — the reward is simply being able to send your transaction
  • Transactions are held in TxPoW units which propagate across the network
  • A TxPoW unit becomes a block if it by chance meets the block difficulty target
  • This ensures all users contribute to chain construction — not just industrial miners

The Burn

The Burn is a small fee incurred when sending transactions during high demand. This fee is permanently destroyed — removed from Minima's hard-capped supply, making Minima deflationary.

The Burn serves three purposes:

  • Incentive to propagate: All users benefit as coins become more scarce
  • Transaction ordering: Higher burn = higher priority (like gas fees)
  • Spam prevention: DoS attacks become expensive; the system self-regulates

MMR Database

To keep the chain small enough for mobile, old blocks are pruned. But we still need to prove coins exist. The Merkle Mountain Range (MMR) solves this.

  • Each coin is stored as a leaf node in a Merkle hash-sum tree
  • A compact proof path from leaf to peak proves any coin exists, even after its block is pruned
  • Users only keep MMR data for their own coins — tiny compared to the full network
  • Everyone stores the MMR peaks and root to verify proofs from others
  • When spending, users present an up-to-date proof that their coin is unspent
This is what makes Minima viable on mobile — you don't need hundreds of gigabytes of chain history.
