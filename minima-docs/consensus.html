Mining & Consensus

How Minima achieves consensus without dedicated miners

Magic Numbers

Four adaptive network parameters that future-proof Minima as devices improve:

ParameterDefault
MaxTxPoWSize64 KB per TxPoW unit
MaxKISSVMOps1,024 script operations
MaxTxnPerBlock256 transactions
MinTxPoWWork1M hashes/second

Recalculated every block via a 16383:1 weighted average. The entire network must agree on a new value — convergence takes ~50 days.

Difficulty

Target Value = Max Value / Difficulty Value. The lower the target, the harder it is to achieve.

  • Transaction difficulty: ~1 second of work per device (minimum 1M hashes)
  • Block difficulty: adjusts by max 10% up/down to maintain 50-second intervals
  • Super block difficulty: each cascade level is 2x harder than the previous

TxPoW Construction

When you send a transaction, your node builds a TxPoW unit in 10 steps:

  1. Set block height, timestamp, main transaction & witness
  2. Set cascade Super Parents from the current tip
  3. Set transaction difficulty target
  4. Calculate chain speed and adjust block difficulty
  5. Sort mempool transactions by burn amount (highest first)
  6. Validate mempool transactions (size, difficulty, MMR proofs, scripts)
  7. Add valid transactions to the TxPoW body (up to 256)
  8. Calculate hashes for main and burn transactions
  9. Construct updated MMR with new coin proofs
  10. Add MMR root hash and total value to header

Mining

After construction, the TxPoW header is hashed repeatedly with an incrementing nonce until the hash falls below the transaction target. If it also falls below the block target (by chance), the TxPoW becomes a block.

Nodes don't aim for block difficulty — they only check after mining. Blocks are a lucky byproduct of transaction mining.

GHOST Protocol

Minima uses Greedy Heaviest Observed SubTree to select the main chain. Unlike "longest chain" rules, GHOST considers total Proof-of-Work weight across all branches — making it more resilient to attack and enabling faster block times.

After receiving any TxPoW unit, the node recalculates all branch weights and selects the heaviest as the main chain. Transactions on orphaned branches return to the mempool.
