Keys & Signatures

Quantum-resistant cryptography in Minima

Winternitz One-Time Signature Scheme

Minima uses WOTSS — a hash-based signature scheme considered quantum-resistant. Unlike ECDSA (used in Bitcoin/Ethereum), Winternitz signatures cannot be broken by Shor's quantum algorithm.

The hash function is SHA3-256, also quantum-resistant. Each key pair can only be used once securely — reuse increases the risk of the private key being deduced.

Merkle Signature Scheme

To get multiple uses from a single public key, Minima combines Winternitz with MMR hash trees. Many one-time key pairs are stored as leaf nodes, with the tree root serving as a reusable public key.

Tree of Trees

A single tree of 64 keys would only allow 64 signatures. Minima scales this with a 3-level tree of trees:

  • Level 1: 64 keys — each signs the root of a Level 2 tree
  • Level 2: 64 trees × 64 keys — each signs the root of a Level 3 tree
  • Level 3: 64² trees × 64 keys — these sign actual transactions

Total: 64³ = 262,144 one-time-use keys per root public key.

The root public key is computed efficiently from just the Level 1 tree. Lower-level trees are generated on demand.

Addresses

All Minima addresses are Pay-to-Script-Hash (P2SH). Every address is the hash of a script. The default script is:

RETURN SIGNEDBY(RootPublicKey)

This means only the private key holder can spend coins at that address.

Transaction Signatures

A full signature includes:

  1. MMR proof path from root to a Level 1 key + parent-child signature connecting Level 1→2
  2. MMR proof path in Level 2 tree + parent-child signature connecting Level 2→3
  3. MMR proof path in Level 3 tree + the actual transaction signature

Any node can verify the full chain from transaction signature → root public key.
