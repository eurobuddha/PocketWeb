Glossary

Key terms used throughout Minima

The Burn

A transaction fee that is permanently destroyed, making Minima deflationary. Higher burn = higher transaction priority.

Cascading Chain

An unbroken chain of Super Block headers that records cumulative Proof-of-Work without storing every block. Enables heavy pruning.

Coin ID

A unique identifier for each UTxO (unspent transaction output) in the network.

Coloured Coins

Custom tokens and NFTs created by "colouring" a tiny fraction of Minima. Native to the protocol — no smart contract needed.

GHOST

Greedy Heaviest Observed SubTree — Minima's chain selection rule. Picks the branch with the most cumulative Proof-of-Work, not just the longest.

KISS VM

Keep It Simple Scripting — Minima's Turing Complete scripting language for smart contracts. Every coin carries a script that must return TRUE to be spent.

Magic Numbers

Four adaptive network parameters (MaxTxPoWSize, MaxKISSVMOps, MaxTxnPerBlock, MinTxPoWWork) that adjust over time as the network evolves.

Maxima

Minima's encrypted peer-to-peer messaging layer. Every node is a Maxima node. Enables off-chain data exchange.

MiniDapp

A decentralized application built with HTML/CSS/JS that runs on a Minima node. Packaged as .mds.zip files.

MDS

MiniDapp System — the application runtime that provides sandboxed access to blockchain commands, SQL databases, file storage, and inter-MiniDapp communication.

MMR

Merkle Mountain Range — an append-only tree structure used to prove coin existence after chain pruning. Users keep only proofs for their own coins.

NFT

A coloured coin with 0 decimal places — indivisible, representing a unique asset.

Omnia

Minima's Layer 2 scaling solution using ELTOO-based payment channels. Enables essentially unlimited transactions per second off-chain.

P2SH

Pay-to-Script-Hash — all Minima addresses are the hash of a script. The script determines spending conditions.

Pulse

A periodic proof-of-work contribution that keeps nodes in sync with the network, even when not sending transactions.

Super Block

A block whose hash exceeds the difficulty of its cascade level by a factor of 2. Stored in the Cascading Chain as proof of cumulative work.

TxPoW

Transaction Proof-of-Work — every user mines their own transaction (~1 second of hashing). No dedicated miners, no block rewards.

UTxO

Unspent Transaction Output — the fundamental unit of value in Minima (same model as Bitcoin). Must be spent in whole.

Winternitz

One-Time Signature Scheme used by Minima. Hash-based and quantum-resistant. Combined with MMR trees to allow multiple signatures per root key.
